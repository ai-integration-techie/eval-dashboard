Metric glossary
True Positive (TP)
The model predicted a label AND it was correct. A direct hit. Higher is always better.
False Positive (FP)
The model predicted a label but it was wrong — a false alarm. Erodes trust and wastes downstream effort.
False Negative (FN)
The model missed a real case — should have predicted but didn't. For support use cases, often worse than FP. A missed Billing or Bug Report goes unaddressed.
True Negative (TN)
Correctly did NOT predict a class for a case that wasn't that class. Warning: TN inflates automatically on small datasets with rare classes. With 10 cases and 8 categories, TN=9 is nearly automatic. Do not use as primary metric.
Precision
Of all times the model predicted class X, how often was it right? Use when false alarms are costly.
TP ÷ (TP + FP)
Recall
Of all actual class X cases, how many did the model catch? Use when missing a case is costly — billing, crashes, security.
TP ÷ (TP + FN)
F1 Score
Harmonic mean of Precision and Recall. If either is bad, F1 suffers. Use as your primary accuracy signal. Scale: 90-100% production ready · 70-89% acceptable · 50-69% needs work · below 50% not ready.
2 × P × R ÷ (P + R)
Macro F1
Simple average of per-class F1 scores. Treats all classes equally — best for revealing zero-recall categories that a weighted average would hide.
Hallucination rate
Percentage of AI outputs containing fabricated or unsupported facts. Critical for RAG, summarization, factual Q&A. Target: below 2-5% depending on app type.
Confidence calibration
How well stated confidence matches actual accuracy. A well-calibrated model that says "90% confident" is right 90% of the time. Measured as |stated confidence − actual accuracy|.
LLM-as-judge
Using a more capable model (GPT-4) to score another model's outputs against a rubric. Scales human evaluation. Scores 1-5 per dimension. Validate by checking agreement with human annotators using Cohen's Kappa (target: above 0.6).
ROUGE / BLEU
Text overlap metrics for generation tasks. ROUGE measures recall (good for summarization). BLEU measures precision (good for translation). Neither captures semantic meaning — pair with embedding similarity.
